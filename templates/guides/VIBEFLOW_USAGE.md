## Quick start

### Run the full workflow

Select the `workflow-orchestrator` agent and provide the input path for the workflow. The orchestrator will:
1. Check for an existing checkpoint (resume) or start fresh.
2. Ask for input paths and confirmation.
3. Dispatch each phase in order to its sub-agent.
4. Track progress in `.vibeflow/workflow-checkpoint.json`.
5. Resume from checkpoint on interruption.

### Run a single phase

```bash
vf dispatch phase-<name>
```

Runs one phase independently with its sub-agent and skill.

## Checkpoint & Resume

The orchestrator creates `.vibeflow/workflow-checkpoint.json` at runtime:

```json
{
  "project": "{{PROJECT_NAME}}",
  "feature": "...",
  "currentPhase": 2,
  "phases": [
    {"name": "Design", "status": "completed", "output": {...}},
    {"name": "Implement", "status": "running", "input": {...}, "output": {...}},
    {"name": "Test", "status": "pending"}
  ]
}
```

To resume after interruption, run again. The orchestrator reads the checkpoint, skips completed phases, and continues from the first non-completed phase.

## Tips

- **Edit agent templates** in `templates/agents/` — re-run `vf init` to regenerate.
- **Edit skill templates** in `templates/skills/` — re-run `vf init` to regenerate.
- **Edit `WORKFLOW_STATE.json`** to retarget phases without regenerating agents.
- **Read `.vibeflow/knowledge/log.md`** to see what previous sessions did.
- Run `vf --help` for all available commands.
