# Naming and Branding

## Chosen name

**VibeFlow**

VibeFlow is short, memorable, and easy to explain:

```text
Vibe = natural developer intent / vibe coding
Flow = structured workflow, agents, skills, verification
```

The name communicates that the tool turns an informal user prompt into a controlled AI software delivery workflow.

## CLI command

Recommended command:

```bash
vf
```

Recommended npm package:

```bash
npx @vibeflow/cli
```

Global install:

```bash
npm install -g @vibeflow/cli
vf
```

## Command style

Keep commands short and memorable:

```bash
vf              # open local web UI
vf init         # initialize repo AI workflow files
vf doctor       # check required tools and permissions
vf run          # run selected engine from current workflow
vf run claude   # dispatch Claude Code
vf run codex    # dispatch Codex CLI
vf run copilot  # dispatch GitHub Copilot CLI
vf skills       # manage skills
vf hooks        # inspect hook status
vf verify       # verify latest output/diff
```

## Positioning

One-line pitch:

```text
VibeFlow turns prompts into verified multi-agent coding workflows across Claude Code, Codex, and Copilot CLI.
```

Short description:

```text
A local-first npm tool that opens a visual AI workflow UI, resolves skills/docs, generates agent instructions, dispatches Claude/Codex/Copilot, verifies results, and evolves skills from real project lessons.
```

## Naming alternatives considered

```text
SkillFlow   - clear but less viral
TaskWeave   - good metaphor but less direct
AgentFlow   - descriptive but generic
CodeWeave   - good but sounds code-only
FlowForge   - strong but heavier enterprise feel
```

VibeFlow is recommended because it is simple, developer-friendly, and works well with a two-letter command: `vf`.
