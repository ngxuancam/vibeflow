## 3. The standard work loop

Apply to every task — from a one-line bug fix to a multi-file feature.

```
┌──────────────────────────────────────────────────────────────────┐
│ 1. SYNC CONTEXT                                                 │
│    If you just edited .vibeflow/* → run: vf init                │
│    (regenerates engine context files from canonical sources)    │
├──────────────────────────────────────────────────────────────────┤
│ 2. SHAPE THE WORK                                               │
│    - Single concern: just run it, no ceremony                   │
│    - Multiple concerns with disjoint file scopes:               │
│        vf units add <name-a>                                    │
│        vf units add <name-b>                                    │
│      (orchestrator auto-serialises overlapping scopes,         │
│       runs disjoint ones in parallel)                           │
├──────────────────────────────────────────────────────────────────┤
│ 3. DISPATCH                                                     │
│    vf orchestrate --engine <claude|codex|copilot> [--yes]       │
│    → plan → spawn workers → collect evidence → independent review│
├──────────────────────────────────────────────────────────────────┤
│ 4. VERIFY before claiming "done"                                │
│    vf verify                                                     │
│    → typecheck + lint + test + confidence≥1 + evidence + scope  │
│    If it fails, do NOT claim done — loop back to step 2         │
├──────────────────────────────────────────────────────────────────┤
│ 5. LOG IT                                                       │
│    Append .vibeflow/knowledge/log.md + update index.md          │
└──────────────────────────────────────────────────────────────────┘
```

### 3.1. Non-negotiable rules
- **No evidence, no conclusion.** Every "I fixed X" / "I tested Y" must cite a file path, command output, or test result.
- **No `vf verify` pass, no completion.** That's the only confidence gate.
- **Don't hand-edit generated files** (AGENTS.md, CLAUDE.md, …). Edit `.vibeflow/*` and re-run `vf init`.
- **Knowledge log is append-only.** Editing past entries breaks the audit trail.

## Appendix: Quick troubleshooting

| Symptom | Likely cause | Fix |
|---------|--------------|-----|
| `vf orchestrate` says "no work units" | No `vf units add` ran, or scope is empty | `vf units status` then create one |
| `vf verify` fails the `confidence<1` gate | A unit hasn't reached confidence 1.0 with evidence | Read `.vibeflow/dispatch/<unit>/review.json`, fix, re-run |
| Engine reads the wrong rules | Hand-edited `AGENTS.md` / `CLAUDE.md` | Edit `.vibeflow/*` and re-run `vf init` |
| `codegraph` reports `indexPresent=false` | `codegraph.db` marker missing | `vf tools enable codegraph` then `vf doctor --probe` |
| `vf skills resolve` finds no skills | Stack doesn't match any skill in `.vibeflow/skills/` | Author a new one from `.claude/skills/_template_/SKILL.md` |
